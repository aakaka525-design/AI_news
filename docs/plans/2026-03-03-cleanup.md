## Task 1: 迁移 legacy_archive 依赖

5 个文件仍从 legacy_archive 导入，需要先将实际代码迁移到 `src/` 下再删除。

**Files:**
- Modify: `rss_fetcher.py` (根目录 compat wrapper)
- Modify: `fetchers/main_money_flow.py`
- Modify: `fetchers/integrity_checker.py`
- Modify: `fetchers/valuation.py`
- Modify: `fetchers/trading_calendar.py`
- Read: 对应的 `legacy_archive/` 源文件

**Step 1: 将 legacy 模块代码复制到 src/ 下**

将以下 legacy 模块内容复制为独立文件：
- `legacy_archive/dashboard/rss_fetcher.py` → 内联到根目录 `rss_fetcher.py`
- `legacy_archive/fetchers/main_money_flow.py` → 内联到 `fetchers/main_money_flow.py`
- `legacy_archive/fetchers/integrity_checker.py` → 内联到 `fetchers/integrity_checker.py`
- `legacy_archive/fetchers/valuation.py` → 内联到 `fetchers/valuation.py`
- `legacy_archive/fetchers/trading_calendar.py` → 内联到 `fetchers/trading_calendar.py`

每个文件：读取 legacy 源码 → 替换 compat wrapper（`from legacy_archive... import *`）为实际代码。

**Step 2: 迁移 api/main.py 的根 compat 导入**

将 `api/main.py` 中的导入路径直接指向 src/：

rss_fetcher 和 sentiment_analyzer 的 lazy import（在函数内部）也需更新。

